# 01 · 프로젝트 기획서

## 1. 서비스 한 줄

> **Murun (무런)** 은 애니뮤 러닝 소모임이 매주 진행하는 정기 운동을 한 화면에서 기록하고, 시즌이 쌓일수록 아카이브로 자라나는 내부용 웹 서비스다.

## 2. 문제 정의

현재 러닝 소모임은 정기 운동 후 기록을 다음과 같이 흩어진 형태로 남기고 있다.

- 운영진의 메모장 / 카카오톡 공지
- 개인 스트라바·런데이 앱(접근권한이 본인만)
- 단체 사진은 카톡 사진첩에 흘러감

시즌이 끝나거나 운영진이 바뀌면 기록이 사실상 소실되고, 신입 부원이 "지난 시즌 우리 클럽이 어땠는지" 확인할 방법이 없다.

## 3. 타깃 사용자

- **운영진(주 기록자)**: 운동 종료 후 5분 안에 그날의 기록을 입력해야 한다. 모바일에서 빠르게.
- **일반 부원(주 조회자)**: 내가 참여한 활동, 누적 거리, 페이스 추이를 가볍게 확인한다.
- **신입/예비 부원(잠재 조회자)**: 클럽 활동 분위기를 사진과 기록으로 확인한다. (공개 범위는 운영 정책에 따라 결정)

## 4. 핵심 가설

1. 입력이 1분 안에 끝나야 운영진이 매주 지속적으로 기록한다.
2. "내 누적 거리" 같은 개인 통계가 한 칸이라도 보이면 부원이 다시 들어온다.
3. 단체사진 1장이 텍스트 10줄보다 아카이브 가치가 크다.

## 5. MVP 범위 (Week 2 종료 시점)

다음만 동작하면 MVP로 본다.

- [ ] 로그인 (Supabase 매직링크 또는 OAuth 1종)
- [ ] **세션 등록**: 날짜·장소·시간·날씨·단체사진 업로드
- [ ] **참여자 기록 등록**: 세션 내에서 N명 × (거리, 기록, 메모) 일괄 입력
- [ ] **세션 상세 페이지**: 사진 + 참여자 표 + 페이스 자동 계산
- [ ] **세션 아카이브 리스트**: 최신순, 카드형
- [ ] 모바일(375px) 정상 동작
- [ ] Vercel에 배포되어 있고 URL 공유 가능

## 6. Week 3 확장 범위 (Stretch)

- [ ] 멤버 페이지: 개인 누적 거리, 페이스 추이 차트
- [ ] 시즌/월별 통계
- [ ] 검색·필터(장소, 멤버)
- [ ] 코멘트 또는 가벼운 반응(👏)
- [ ] PWA 설치 / 오프라인 폴백
- [ ] 공개/비공개 토글

## 7. 비목표 (Non-goals, 3주 안에는 안 한다)

- 실시간 GPS 트래킹, 스트라바 연동
- 푸시 알림 / 이메일 알림
- 다국어
- 운영진 권한 세분화(role 1단계만)
- 결제, 회비 관리

## 8. 성공 지표 (3주 끝나는 시점에 스스로 평가)

| 지표 | 목표 |
|------|------|
| 실제 시즌 기록 입력 횟수 | ≥ 3건 (지난 3회 운동) |
| 모바일에서 세션 1건 입력 소요 시간 | ≤ 90초 |
| Lighthouse Performance (모바일) | ≥ 80 |
| 빌드·배포 자동화 | push to main → vercel auto deploy |
| 내가 만든 워크플로우로 구현한 feature 비율 | ≥ 70% |

## 9. 리스크와 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|

| 인증 흐름이 깊어져 MVP가 늦어짐 | 일정 지연 | Week 2 초반에는 "운영진 1계정 + 비밀번호" 같은 단순 형태로도 OK |
